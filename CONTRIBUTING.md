# Contributing to RenameConv

First of all, thank you for your interest in contributing to RenameConv!

Contributions of all kinds are welcome, including bug reports, feature requests, documentation improvements, and code contributions.

---

## How to Contribute

### Reporting Bugs

Before opening a bug report:

- Make sure you're using the latest release.
- Search existing Issues to avoid duplicates.
- Include as much information as possible.

---

### Suggesting Features

Feature requests are welcome.

When suggesting a feature, please explain:

- The problem it solves.
- Why it would be useful.
- Any alternative solutions you've considered.

---

### Pull Requests

Before submitting a Pull Request:

- Keep changes focused and minimal.
- Follow the existing coding style.
- Test your changes before submitting.
- Update documentation if necessary.

---

## Development

RenameConv is written in **C#** and uses **FFmpeg** for media conversion.

If your contribution introduces new functionality, please ensure it does not break existing behavior.

---

## Code Style

- Write clear and readable code.
- Use meaningful names.
- Keep methods small whenever possible.
- Remove unused code before submitting.

---

## Questions

If you're unsure about an idea or implementation, feel free to open an Issue before starting work.

---

Thank you for helping make RenameConv better! ❤️
